<td colspan="2"><span class="postbody">Apache httpd 2.4.38 is released as GA.
<br>
<span style="font-weight: bold">
<br>
<span style="color: green">28 February 2019 - Update dependencies , see below </span>
<br>
</span>
<br>
ASF and Apachelounge changes :
<br>

<br>
<span style="font-weight: bold"> <a href="http://www.apachelounge.com/Changelog-2.4.html" target="_blank">www.apachelounge.com/Changelog-2.4.html</a> </span>
<br>

<br>
Build with dependencies:
<br>

<br>
 - VC15 openssl 1.1.1b, VC14 1.0.2r 
<br>
 - nghttp2 1.36.0
<br>
 - jansson 2.12
<br>
 - curl 7.64.0
<br>
 - apr 1.6.5 
<br>
 - apr-util 1.6.1
<br>
 - apr-iconv 1.2.2
<br>
 - zlib 1.2.11 
<br>
 - brotli 1.0.7
<br>
 - pcre 8.43
<br>
 - libxml2 2.9.9
<br>
 - lua 5.2.4
<br>
 - expat 2.2.6
<br>

<br>
<span style="font-weight: bold">VC15 notes:</span>
<br>
VC15 is backward compatible to VC14. That means, a VC14 module can be used inside a VC15 binary (for example PHP VC14 as module). Because this compatibility the version number of the Redistributable is 14.1x.xx and after you install, the Redistributable VS2015 is updated from 14.0x.xx to VS2017 14.1x.xx (you can still use VC14).
<br>

<br>
Documentation: <a href="http://httpd.apache.org/docs/2.4/" target="_blank">http://httpd.apache.org/docs/2.4/</a>
<br>

<br>
When you have hangs, slow traffic and/or when having in your log entries like Asynchronous AcceptEx failed. You can try the following settings: 
<br>

<br>
AcceptFilter http none 
<br>
AcceptFilter https none 
<br>
EnableSendfile off 
<br>
EnableMMAP off 
<br>

<br>
Enjoy, 
<br>

<td colspan="2"><span class="postbody">The ASF forgot to mention security vulnerabilities fixed in the initial changelog 2.4.38. 
<br>

<br>
Added now to<span style="font-weight: bold"> <a href="http://www.apachelounge.com/Changelog-2.4.html" target="_blank">www.apachelounge.com/Changelog-2.4.html</a>  </span>
<br>

<br>
  *) SECURITY: CVE-2018-17199 (cve.mitre.org)
<br>
     mod_session: mod_session_cookie does not respect expiry time allowinesessions to be reused.  [Hank Ibell]
<br>

<br>
  *) SECURITY: CVE-2018-17189 (cve.mitre.org)
<br>
     mod_http2: fixes a DoS attack vector. By sending slow request bodiesto resources not consuming them, httpd cleanup code occupies a serverthread unnecessarily. This was changed to an immediate stream resetwhich discards all stream state and incoming data.  [Stefan Eissing]
<br>

<br>
  *) SECURITY: CVE-2019-0190 (cve.mitre.org)
<br>
     mod_ssl: Fix infinite loop triggered by a client-initiated
<br>
     renegotiation in TLSv1.2 (or earlier) with OpenSSL 1.1.1 and
<br>
     later.  PR 63052.  [Joe Orton]
<br>

<br>
For details see<span style="font-weight: bold"> <a href="https://httpd.apache.org/security/vulnerabilities_24.html" target="_blank">https://httpd.apache.org/security/vulnerabilities_24.html</a> </span></span><span class="gensmall"></span></td>

<td colspan="2"><span class="postbody">This one is important, advise to upgrade !
<br>

<br>
<span style="font-weight: bold"><span style="color: red">CVE-2019-0190</span></span> : mod_ssl 2.4.37 remote DoS when used with OpenSSL 1.1.1
<br>

<br>
Severity: Important
<br>

<br>
Vendor: The Apache Software Foundation
<br>

<br>
Versions Affected:
<br>
httpd 2.4.37
<br>

<br>
Description:
<br>
A bug exists in the way mod_ssl handled client renegotiations.A remote attacker could send a carefully crafted request that would cause mod_ssl to enter a loop leading to a denial of service. This bug can be only triggered with Apache HTTP Serverversion 2.4.37 when using OpenSSL version 1.1.1 or later, due to an interaction in changes to handling of renegotiation attempts.
<br>

<br>
Mitigation:
<br>
All httpd users consuming mod_ssl combined with OpenSSL 1.1.1 or later should upgrade to 2.4.38 or later.
<br>

<br>
Credit:
<br>
The issue was identified through user bug reports.</span><span class="gensmall"><br><br>Last edited by admin on Wed 23 Jan '19 13:16; edited 4 times in total</span></td>
